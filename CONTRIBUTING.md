# Contributing Guidelines
 
> **Project:** SithCare – Mental Health Channelling Platform
> **Module:** SENG 31242 – System Design Project
> **University:** University of Kelaniya
> **Group:** Group 10
 
Thank you for contributing to the SithCare Prototypes Repository. This repository manages all UI/UX prototype artefacts related to the SithCare Mental Health Channelling Platform.
 
---
 
## Contribution Workflow
 
### 1. Create or Assign an Issue
 
Before starting any work:
 
- Create or assign a GitHub Issue
- Add appropriate labels and milestones
- Define clear acceptance criteria
---
 
### 2. Create a Branch
 
Follow the branch naming conventions below:
 
```text
draft/<feature-name>
fix/<issue-number>
```
 
**Examples:**
 
```
draft/login-wireframe
draft/emergency-flow
fix/22-dashboard-layout
```
 
---
 
### 3. Make Changes
 
When updating the repository:
 
- Commit editable source files
- Commit exported PNG / SVG / PDF files
- Organise files into the correct folders
---
 
## File Naming Conventions
 
### Wireframes
 
```
wf-<module-name>-v1.png
```
 
> Example: `wf-patient-dashboard-v1.png`
 
### User Flows
 
```
uf-<flow-name>.drawio
```
 
> Example: `uf-emergency-booking.drawio`
 
### Mockups
 
```
mockup-<screen-name>.fig
```
 
---
 
## Commit Message Standard
 
Use the [Conventional Commits](https://www.conventionalcommits.org/) format.
 
**Good examples:**
 
```
docs(prototypes): add appointment booking wireframe
feat(mockup): create clinic admin dashboard
fix(user-flow): correct telemedicine navigation
```
 
**Avoid unclear messages such as:**
 
```
updated
ui changes
final version
new file
```
 
---
 
## Pull Request Checklist
 
Before submitting a Pull Request, verify the following:
 
- [ ] Files are correctly organised into the right folders
- [ ] Source files are included
- [ ] Exported assets are included
- [ ] Naming conventions are followed
- [ ] Related issue is linked
- [ ] Peer review has been requested
---
 
## Design Standards
 
All contributions should follow these design principles:
 
- Keep interfaces simple and accessible
- Maintain consistent spacing and layouts
- Consider elderly-friendly usability
- Support multilingual design concepts
- Use calm and user-friendly visual design principles
---
 
## Review Process
 
- A minimum of **one approval** is required before merging
- All review comments must be resolved before merge
- Merge only after full verification by the reviewer
